## Examples

### Default

```jsx live
<NavList aria-label="Docs navigation">
  <NavList.Item defaultExpanded>
    GitHub Copilot
    <NavList.SubNav>
      <NavList.Item href="#" aria-current="page">
        Overview
      </NavList.Item>
      <NavList.Item href="#">Concepts</NavList.Item>
      <NavList.Item href="#">Quickstart</NavList.Item>
    </NavList.SubNav>
  </NavList.Item>
  <NavList.Item href="#">Code review</NavList.Item>
</NavList>
```

### Sections

Use first-level `NavList.Item` rows as sections. Adjacent sections include a divider, and each section can optionally expand to reveal nested items.

```jsx live
<NavList aria-label="Section navigation">
  <NavList.Item defaultExpanded>
    Products
    <NavList.SubNav>
      <NavList.Item href="#" aria-current="page">
        Copilot
      </NavList.Item>
      <NavList.Item href="#">Codespaces</NavList.Item>
    </NavList.SubNav>
  </NavList.Item>
  <NavList.Item defaultExpanded>
    Collaboration
    <NavList.SubNav>
      <NavList.Item href="#">Pull requests</NavList.Item>
      <NavList.Item href="#">Branches</NavList.Item>
    </NavList.SubNav>
  </NavList.Item>
</NavList>
```

### Four levels

NavList supports up to four levels. Levels one through three can optionally expand or collapse; level four is reserved for leaf article links.

```jsx live
<NavList aria-label="Four-level navigation">
  <NavList.Item defaultExpanded>
    GitHub Copilot
    <NavList.SubNav>
      <NavList.Item defaultExpanded>
        Guides
        <NavList.SubNav>
          <NavList.Item defaultExpanded>
            Agents
            <NavList.SubNav>
              <NavList.Item href="#" aria-current="page">
                Build an agent
              </NavList.Item>
              <NavList.Item href="#">Customize an agent</NavList.Item>
            </NavList.SubNav>
          </NavList.Item>
          <NavList.Item href="#">Extensions</NavList.Item>
        </NavList.SubNav>
      </NavList.Item>
      <NavList.Item href="#">Quickstart</NavList.Item>
    </NavList.SubNav>
  </NavList.Item>
  <NavList.Item href="#">Code review</NavList.Item>
</NavList>
```

### Nested navigation

Use `NavList.SubNav` inside a `NavList.Item` for a nested list. Items with nested lists render as full-row buttons with a triangle indicator, `aria-expanded`, and `aria-controls`.

```jsx live
<NavList aria-label="Nested navigation">
  <NavList.Item defaultExpanded>
    Documentation
    <NavList.SubNav>
      <NavList.Item href="#" aria-current="page">
        Overview
      </NavList.Item>
      <NavList.Item href="#">Guides</NavList.Item>
      <NavList.Item href="#">API reference</NavList.Item>
    </NavList.SubNav>
  </NavList.Item>
  <NavList.Item href="#">Changelog</NavList.Item>
</NavList>
```

### Custom links

Pass a custom link component with the `as` prop when routing frameworks need to control navigation. The component must render an accessible link.

```jsx live
const CustomLink = React.forwardRef(({children, ...props}, ref) => (
  <a ref={ref} data-custom-link="true" {...props}>
    {children}
  </a>
))

render(
  <NavList aria-label="Custom link navigation">
    <NavList.Item as={CustomLink} href="#">
      Custom link
    </NavList.Item>
  </NavList>,
)
```

## Accessibility

`NavList` renders a `nav` landmark containing unordered lists. Provide either `aria-label` or `aria-labelledby` when the surrounding page has more than one navigation landmark.

Use `aria-current` on the current `NavList.Item`; this is the current-item contract. `aria-current="page"` is recommended for page navigation, but any valid `aria-current` value is passed through. A parent item with a current descendant automatically opens its nested `NavList.SubNav`.

Items with `NavList.SubNav` render as disclosure buttons instead of links. The button exposes `aria-expanded` and `aria-controls`, and pressing <kbd>Escape</kbd> while focus is inside an expanded nested list collapses it and returns focus to the parent item.

The top collapse bar, sticky sidebar behavior, and responsive page shell behavior are out of scope for `NavList`. Compose those behaviors at the page or layout level if needed.

## Component props

### NavList <Label>Required</Label>

| name                       | type        | default        | required | description                                                  |
| -------------------------- | ----------- | -------------- | -------- | ------------------------------------------------------------ |
| `children`                 | `ReactNode` |                | `true`   | `NavList.Item` and `NavList.Divider` nodes                   |
| `aria-label`               | `string`    | `'Navigation'` | `false`  | Accessible label for the navigation landmark                 |
| `aria-labelledby`          | `string`    |                | `false`  | ID of the element that labels the navigation landmark        |
| `className`                | `string`    |                | `false`  | Sets a custom class on the `nav` element                     |
| `data-testid`              | `string`    |                | `false`  | Sets a custom test id on the `nav` element                   |
| `internalAccessibleLabels` | `object`    |                | `false`  | Customizes labels for internally rendered controls           |

### NavList.Item <Label>Required</Label>

| name               | type                       | default | required | description                                                        |
| ------------------ | -------------------------- | ------- | -------- | ------------------------------------------------------------------ |
| `as`               | `React.ElementType`        | `'a'`   | `false`  | Custom link component or element                                   |
| `children`         | `ReactNode`                |         | `true`   | Item label and optional `NavList.SubNav`                           |
| `href`             | `string`                   |         | `false`  | URL for leaf links when rendering as an anchor                     |
| `aria-current`     | <NavListAriaCurrentProp /> |         | `false`  | Marks the item as current                                          |
| `defaultExpanded`  | `boolean`                  | `false` | `false`  | Opens nested `NavList.SubNav` by default                           |
| `expanded`         | `boolean`                  |         | `false`  | Controls the nested expanded state                                 |
| `onExpandedChange` | `(expanded) => void`       |         | `false`  | Called when the nested expanded state changes                      |
| `leadingVisual`    | `ReactElement \| Icon`     |         | `false`  | Visual rendered before the label                                   |
| `trailingVisual`   | `ReactElement \| Icon`     |         | `false`  | Visual rendered after the label                                    |
| `disabled`         | `boolean`                  | `false` | `false`  | Adds `aria-disabled`, prevents clicks, and applies disabled styles |

### NavList.SubNav

| name        | type        | default | required | description                            |
| ----------- | ----------- | ------- | -------- | -------------------------------------- |
| `children`  | `ReactNode` |         | `true`   | Nested `NavList.Item` nodes            |
| `className` | `string`    |         | `false`  | Sets a custom class on the nested `ul` |

### NavList.Divider

`NavList.Divider` renders a separator between sections or items.

| name        | type     | default | required | description                          |
| ----------- | -------- | ------- | -------- | ------------------------------------ |
| `className` | `string` |         | `false`  | Sets a custom class on the separator |
